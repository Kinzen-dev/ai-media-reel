How a letter is built

A skeleton of stroke paths is scaled and jittered per work, baked into a cached Path2D, then stroked. The skeleton never changes; the metrics reshape it. That is why every title looks like its own typeface.
generation-time
Stroke weight. A slow, expensive render thickens every stroke; a fast model draws thin and wiry. Heavy ink = heavy compute.
token count
Letter width. A verbose prompt stretches each glyph wide; a terse one condenses it. The headline literally widens with its prompt.

seed
Control-point jitter. The seed seeds a deterministic wobble on every bezier handle, so the same word from a different seed is a different hand.
