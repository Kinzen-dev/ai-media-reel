Typed @property gradient system

SPECTRUM

A brand identity that is never the same color twice.

Enter the field->
oklch hue A 250 hue B 190 axis 0

The gradient is not a background. It is the brand, and it is registered, typed, and alive.

The mechanic
@property
@property --hue-a {
  syntax: "<number>";
  inherits: false;
  initial-value: 250;
}
Why it works
It interpolates
Unregistered custom props are opaque strings. Registration gives the browser a TYPE, so --hue-a and --axis tween between keyframes. No JS needed for the core animation.
The palette
oklch
oklch(L C H) over a near-black field. Only the hue H sweeps, so chroma and lightness hold the brand on-palette while every frame is a new color.

Static fallback: browsers without @property render the flat brand color and never see a broken half-animated gradient. prefers-reduced-motion freezes the field at a single chosen oklch stop and shows the final, fully-formed identity. Same mechanic on mobile - no cropping, the gradient is resolution-independent. Build native CSS, no GSAP, no canvas, no WebGL.
